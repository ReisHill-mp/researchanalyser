# Research Analyser Workflow Branching

## Step 1. Project setup
- Project name
- Short description / study objective

## Step 2. Test type
Choose one:
- Unmoderated test
- Unmoderated with balanced comparison
- Moderated

## Unmoderated test workflow
1. Project setup
2. Test type
3. Research script
4. Collect transcripts
5. Question-by-question analysis
6. Final report

## Unmoderated with balanced comparison workflow
1. Project setup
2. Test type
3. Research script
4. Collect transcripts
5. Question-by-question analysis
6. Final report

Notes:
- Balanced participant ordering lives in transcript collection because it depends on the collected participants.
- The analysis output should show:
  - Balanced comparison
  - Group A questions
  - Group B questions
  - Other questions

## Moderated workflow
1. Project setup
2. Test type
3. Research inputs
4. Collect transcripts
5. Research analysis
6. Final report

Notes:
- Moderated studies do not rely on a strict question-by-question artifact as the main user-facing analysis view.
- The analysis should be theme-led and grounded in the research guide, upfront context, and transcripts.

## Product intent
- Keep the overall product simple.
- Branch the wording and analysis model by study type rather than forcing one structure onto every study.
- Reuse the same project, transcript, analysis, and report storage model where possible.
